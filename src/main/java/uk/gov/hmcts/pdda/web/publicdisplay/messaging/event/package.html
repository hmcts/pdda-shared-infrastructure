This package consists of the event store related classes used by public display.

EventStore defines a common interface for classes that can provide buffering functionality for the events. The method pushEvent is called to push an event to the store and the method popEvent is used to pop an event from the store.

EventStoreFactory class is used to create event store instances. The event store type should be specified using the property public.display.event.store.type, which is the fully qualified class name of the event type. If the property is not specified the type that is created is uk.gov.hmcts.pdda.web.publicdisplay.messaging.event.DefaultEventStore.

DefaultEventStore provides a FIFO event store implementation. If the store is empty, the thread that calls the popEvent method will wait by releasing the monitor lock, so that other threads can push events to the store.
