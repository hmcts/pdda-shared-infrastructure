This package contains the classes that represent the types of configuration changes and initialisations available to public displays. The super-class is CourtConfigurationChange, this type of change is intended to be the fail safe, should the configuration component not know how to handle a sub-class, it should treat it as a configuration change for the whole court.

The sub-classes of CourtConfigurationChange represent the configuration changes available to a user of the Public Display system. A CourtDisplayConfigurationChange represents a configuration change to the a display and a CourtRotationSetConfigurationChange is a configuration change to a rotation set.

For the purposes of initialisation, all types of CourtConfigurationChange support the concept of 'recreate', which tells the configuration component to produce as render changes all elements associated with the scope of the CourtConfigurationChange. If recreate is not selected, then only the delta between original and current configuration is produced as render changes. By default CourtConfigurationChange has recreate selected and its sub-classes do not.
